Backend QA audit complete. Report written to `QA_REPORT_BACKEND.md`.

**Key findings — 24 bugs total:**

**4 Critical:**
- **B-001:** `social_auth.tokens` has NO RLS — all 26 OAuth tokens (access_token, refresh_token) exposed to anyone with the anon key
- **B-002:** `GET /api/v2/accounts` returns ALL users' accounts (no `user_id` filter)
- **B-003:** Sync route returns `access_token` in JSON response via `select('*')`
- **B-004:** Cron secret validation is bypassable — if `CRON_SECRET` env var is unset, anyone can trigger post publishing

**7 High:** Unauthenticated routes (audio, pexels, chat, batch-export), debug_logs without RLS, no account ownership checks, token cache cross-user leakage, Facebook token prefix logging

**8 Medium:** No rate limiting on paid AI APIs, tokens in URL params, no pagination, timezone bugs in scheduling, no idempotency, unauthenticated proxy

**5 Low:** Defense-in-depth gaps, inconsistent schema usage, broken upsert conflict key

The most urgent action is enabling RLS on `social_auth.tokens` — this is a live credential exposure affecting all connected social accounts.
